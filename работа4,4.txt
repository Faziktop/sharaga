## Основная часть (оформлена строго по ГОСТ 19.505-79)

### 1. НАЗНАЧЕНИЕ ПРОГРАММЫ

Программное изделие **«Клиник-Онлайн»** предназначено для автоматизации процессов записи пациентов на приём к врачам медицинской клиники, управления рабочим расписанием врачей и выполнения административных задач, связанных с приёмом.

Функции программы в части работы оператора (администратора):
- регистрация пациентов в системе;
- запись пациента на приём;
- изменение и отмена записи;
- просмотр и редактирование расписания врачей;
- формирование daily-листа (список пациентов на день);
- формирование отчётов по загрузке врачей и посещаемости.

Данных, приведённых в разделе, достаточно для понимания функционального назначения программы.

---

### 2. УСЛОВИЯ ВЫПОЛНЕНИЯ ПРОГРАММЫ

#### 2.1. Минимальный состав аппаратурных средств:
- Процессор: 2 ядра, 2 ГГц
- ОЗУ: 4 ГБ
- Диск: 20 ГБ свободного места
- Сетевое устройство (Ethernet/Wi-Fi)

#### 2.2. Максимальный состав аппаратурных средств:
- Процессор: 4 ядра, 3 ГГц и выше
- ОЗУ: 8 ГБ и более
- Диск: SSD, 50 ГБ

#### 2.3. Программные средства:
- Операционная система: Windows 10/11 или Linux (Ubuntu 20.04+)
- Браузер: Google Chrome (актуальная версия), Microsoft Edge
- Доступ в локальную сеть клиники или интернет (HTTPS)
- СУБД: PostgreSQL / MySQL (серверная часть)

Программа функционирует в клиент-серверной архитектуре. Рабочее место оператора должно быть подключено к серверу системы.

---

### 3. ВЫПОЛНЕНИЕ ПРОГРАММЫ

#### 3.1. Загрузка программы

1. Включить компьютер.
2. Запустить браузер (Google Chrome).
3. В адресной строке ввести URL системы, например:  
   `https://clinic-online.healthplus.local`
4. Нажать клавишу **Enter**.
5. В открывшейся форме авторизации ввести:
   - логин (например, `admin1`)
   - пароль
6. Нажать кнопку **«Вход»**.

Результат: открывается главное окно системы с ролью **«Администратор»**.

#### 3.2. Выполнение программы

##### 3.2.1. Запись пациента на приём

| Действие оператора | Ответ программы |
|--------------------|----------------|
| Выбрать раздел «Запись на приём» | Отображается форма поиска врача |
| Выбрать специализацию и врача | Показывается расписание на ближайшие 7 дней |
| Выбрать свободный слот времени | Форма подтверждения записи |
| Ввести ФИО, телефон, дату рождения пациента | Проверка уникальности (поиск дубликатов) |
| Нажать «Записать» | Сообщение: «Запись успешно создана. Пациенту отправлено SMS/email уведомление» |

##### 3.2.2. Формирование daily-листа

| Действие оператора | Ответ программы |
|--------------------|----------------|
| Выбрать раздел «Расписание» → «Daily-лист» | Отображается список врачей |
| Выбрать врача и дату | Формируется список пациентов на день с указанием времени и контактов |
| Нажать «Экспорт» | Выбор формата (PDF / XLSX) и сохранение файла |

##### 3.2.3. Завершение программы

1. Нажать на кнопку **«Выход»** (верхний правый угол).
2. Подтвердить выход в диалоговом окне: **«Вы действительно хотите выйти?»** → кнопка **«Да»**.
3. Закрыть браузер.
4. При необходимости выключить компьютер (по правилам организации).

---

### 4. СООБЩЕНИЯ ОПЕРАТОРУ

#### 4.1. Сообщения в ходе выполнения программы

| Текст сообщения | Действие оператора |
|----------------|--------------------|
| «Соединение с сервером установлено» | Начать работу |
| «Запись успешно создана» | Продолжить работу или перейти к следующей записи |
| «Время уже занято» | Выбрать другое время |
| «Пациент с таким телефоном уже существует. Выберите действие: [Использовать данные] [Изменить]» | Выбрать нужный вариант |
| «Не заполнены обязательные поля: ФИО, телефон» | Заполнить поля |
| «Отчёт сформирован» | Сохранить или распечатать |

#### 4.2. Действия оператора в случае сбоя

| Ситуация | Действие оператора |
|----------|--------------------|
| Программа не отвечает более 30 секунд | Закрыть браузер (через диспетчер задач), заново открыть и авторизоваться |
| Сообщение «Ошибка подключения к БД» | 1. Проверить сетевое подключение. 2. Сообщить системному администратору. 3. Не вносить изменения до восстановления связи |
| Потеря данных при записи (например, зависание) | Проверить раздел «Расписание → Записи»: если запись отсутствует, создать заново |

#### 4.3. Повторный запуск программы после сбоя

1. Закрыть все вкладки браузера.
2. Повторить действия по п. 3.1 (загрузка программы).
3. Проверить целостность данных (daily-лист, записи).
4. При обнаружении расхождений — сверить с журналом регистратуры.
